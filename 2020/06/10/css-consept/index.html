css相关概念

  • 2020-06-10

语法

  • @规则：

    • @charset, 定义样式表使用的字符集.
    • @import, 告诉 CSS 引擎引入一个外部样式表.
    • @namespace, 告诉 CSS 引擎必须考虑XML命名空间。
    • 嵌套@，是嵌套语句的子集,不仅可以作为样式表里的一个语句，也可以用在条件规则组里：
      • @media, 如果满足媒介查询的条件则条件规则组里的规则生效。
      • @page, 描述打印文档时布局的变化.
      • @font-face, 描述将下载的外部的字体。
      • @keyframes, 描述 CSS 动画的中间步骤 .
      • @supports, 如果满足给定条件则条件规则组里的规则生效。
      • @document, 如果文档样式表满足给定条件则条件规则组里的规则生效。 (推延至 CSS Level 4 规范)

  • 重置样式：all: unset

  • 选择器类型的优先级是递增的：
    • 类型选择器（例如，h1）和伪元素（例如，::before）
    • 类选择器 (例如，.example)，属性选择器（例如，[type=”radio”]）和伪类（例如，:hover）
    • ID 选择器（例如，#example）
    • 通配选择符（universal selector）（*）关系选择符（combinators）（+, >, ~, ‘ ‘, ||）和 否定伪类（negation pseudo-class）（:not()）对优先级没有影响。（但是，在 :not() 内部声明的选择器会影响优先级）
    • 给元素添加的内联样式 (例如，style=”font-weight:bold”) 总会覆盖外部样式表的任何样式 ，因此可看作是具有最高的优先级。
    • !important 例外规则：用于覆盖内联样式或者高优先级的样式，应该尽量少使用，要想覆盖一个!important样式可以再声明一个!important加上更高优先级
  • css中（如background-image）,url()的参数可以也可以不使用引号。如果使用引号包围了URL，那么它会被解析为一个，包含对某些字符的额外转义
  • display属性：全局属性值有 inherit|initial|unset
    • display-outside 外部显示类型：block | inline | run-in，指示元素在流式布局中的表现（块级或内联元素）
      • block: 在正常流中产生块元素框，并在元素首尾产生换行符
      • inline: 在正常流中产生一个或多个（ul）内联元素盒子，并且在元素首尾不会产生换行符，如果空间足够，在其后的元素将在同一行显示。
      • run-in: chrom早期版本4-31支持，现在已废弃
  • display-inside 内部显示类型：flow | flow-root | table | flex | grid | ruby，对于非可替换元素，内部显示类型定义格式换上下文如何产生，指示内部元素如何布局，注意可替换元素的内部布局不是css能控制的。
    • flow:外部显示类型为inline或run-in产生行内格式化上下文，否则产生块级格式上下文
    • flow-root: 该元素生成一个块元素框，该框将建立一个新的块格式化上下文 ，以定义格式化根所在的位置.
    • table:
    • flex
    • grid
    • ruby:元素的行为类似于内联元素，并根据ruby格式化模型布置其内容. 它的行为类似于相应的HTML ruby元素.
  • display-listitem : 取值 display-outside? && [ flow | flow-root ]? && list-item

  • display-internal：table-row-group | table-header-group |table-footer-group | table-row | table-cell |table-column-group | table-column | table-caption |
    ruby-base | ruby-text | ruby-base-container |ruby-text-container。某些布局模型（例如table和ruby具有复杂的内部结构，其子代和后代可以担当几种不同的角色. 本部分定义了那些”内部”显示值，它们仅在该特定布局模式下才有意义.

  • display-box ： contents | none ，规定元素是否产生boxs

    • none:视觉上影藏不可见，不占用文档流位置，但屏幕阅读器可读，如果需要占位请改用visibility属性.
    • contents
  • display-legacy：inline-block | inline-table | inline-flex | inline-grid , 同一布局模式的块级和内联级变体,CSS 2为display属性使用了单关键字语法,等效于 外部显示类型-内部显示类型，在display更好地支持两个关键字值之前，开发人员应优先使用它们。

不对浏览器进行任何布局控制时浏览器的默认布局方式

  • 弹性盒

    一维布局，一次只能处理一个维度上的元素布局，一行或者一列，作为对比的是另外一个二维布局 CSS Grid Layout，可以同时处理行和列上的布局。
    flex布局的主轴与交叉轴，主轴由flex-direction定义，交叉轴与主轴垂直。flex-direction取值：

    • row

    • row-reverse

    • column

    • column-reverse

      flex容器内元素默认行为：

    • 元素排列为一行 (flex-direction 属性的初始值是 row)。

    • 元素从主轴的起始线开始。

    • 元素不会在主维度方向拉伸，但是可以缩小。

    • 元素被拉伸来填充交叉轴大小。

    • flex-basis 属性为 auto。

    • flex-wrap 属性为 nowrap。

      flex-flow: flex-direction与flex-wrap的简写属性

      flex容器内元素简写属性flex:flex-grow flex-shrink flex-basis

      几种预定义的简写：

    • flex: inital 把flex元素重置为Flexbox的初始值,相当于flex:0 1 auto

    • flex: auto 相当于 flex: 1 1 auto

    • flex: none 相当于 flex:0 0 auto

    • flex: 数值 相当于flex: 数值 数值 0

      flex-basis: 指定了 flex 元素在主轴方向上的初始大小。flex容器内空间减去元素占用空间就得到剩余空间，flex-basis=0时就相当于将元素宽度让渡给剩余空间供所有容器内元素扩张。如果不使用 box-sizing 改变盒模型的话，那么这个属性就决定了 flex 元素的内容盒（content-box）的尺寸。注意，当一个元素同时被设置了 flex-basis (除值为 auto 外) 和 width (或者在 flex-direction: column 情况下设置了height) , flex-basis 具有更高的优先级，优先级关系：max-width/minwidth>flex-basis>width>内容宽度。

  • 网格

    网格仅仅是由水平和垂直线集合创建的一个模式，我们可以根据这个模式排列我们的设计元素。它们帮助我们创建设计——在页面之间移动时元素不会跳动或更改宽度，从而在我们的网站上提供高一致性。这意味着您不需要考虑某个元素相对于其他元素的宽度。您的选择限于“该元素将跨越多少个网格列”。

    在css grid网格之前，大多数网格框架都是基于float技术实现，从float多列布局演进为网格布局。网格布局一般分为12个列，因为12可以被2346整除，可以适应不同的情况。网格布局的因素有：单列宽度，列沟槽，容器宽度，为了更高的适用性会限制max-width,且各项宽度使用百分比，例如max-width为980px宽度的border-box容器，除去内边距20px，剩余960px的内容宽度布局列和沟槽，如果沟槽设置20px，一共12个沟槽1220 = 240px，列宽度就是（960-240）/12 = 60px，列宽百分比宽度就是60/960 = 6.25%，沟槽宽度百分比就是20/960 = 2.08333333％,这样跨越n列的宽度计算就是 calc((6.25%*n) + (2.08333333%(n-1)))。

    使用浮动技术实现网格布局的限制：
    最大限制是它基本上是一维的。我们处理的是列元素只能跨越多个列，而不能跨越行。
    当元素宽度超过行空间时会溢出
    当跨列宽度超过一行宽度时由于浮动的关系将会换行显示从而破坏网格，所以需要保证行总宽度正确。

    Flexbox是一维设计。它处理单个维度，即行或列。不能为列和行创建严格的网格，这意味着如果我们要为网格使用flexbox，我们仍然需要为浮动布局计算百分比。

概念

  • 边距折叠
  • 块格式化上下文（Block Formatting Context，BFC）Web页面的可视CSS渲染的一部分，是块盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。
块格式化上下文包含创建它的元素内部的所有内容，块格式化上下文对浮动定位（ float）与清除浮动（clear）都很重要。浮动定位和清除浮动时只会应用于同一个BFC内的元素。浮动不会影响其它BFC中元素的布局，而清除浮动只能清除同一BFC中在它前面的元素的浮动。外边距折叠（Margin collapsing）也只会发生在属于同一BFC的块级元素之间。

下列方式会创建块格式化上下文：
- 根元素(<html>)
- 浮动元素（元素的 float 不是 none）
- 绝对定位元素（元素的 position 为 absolute 或 fixed）
- 行内块元素（元素的 display 为 inline-block）
- 表格单元格（元素的 display为 table-cell，HTML表格单元格默认为该值）
- 表格标题（元素的 display 为 table-caption，HTML表格标题默认为该值）
- 匿名表格单元格元素（元素的 display为 table、table-row、 table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot的默认属性）或 inline-table）
- overflow 值不为 visible 的块元素
- display 值为 flow-root 的元素
- contain 值为 layout、content或 paint 的元素
- 弹性元素（display为 flex 或 inline-flex元素的直接子元素）
- 网格元素（display为 grid 或 inline-grid 元素的直接子元素）
- 多列容器（元素的 column-count 或 column-width 不为 auto，包括 column-count 为 1）
- column-span 为 all 的元素始终会创建一个新的BFC，即使该元素没有包裹在一个多列容器中（标准变更，Chrome bug）。

注意事项

  • 尽管用一组选择器来定义的单个规则集，比用多个单一选择器来定义的多个规则集更加简洁，但这却不适用于规则集的有效性。这会导致一个重要的后果：如果其中的一个选择器是无效的，如使用了一个未知的伪元素或伪类，会导致整个选择器都会无效，同时对应的规则都会被忽略。

  • @font-face:

    • 引用远程字体仍然受同域限制，通过HTTP access controls 解除这一限制。
    • 因为这里没有为 TrueType(ttf), OpenType(otf) 和 Web Open File Format(WOFF) 字体定义MIME，因此不能为这些字体类型设置特定的MIME（实际上WOFF的MIME将会是application/font-woff，但浏览器对此MIME的识别还不统一，其它字体情况也类似，可暂时使用application/octet-stream）
    • 不能在一个 CSS 选择器中定义 @font-face ，这样写是无效的。
