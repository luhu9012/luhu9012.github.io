布局视口（layout viewport) 与 可视视口（vitrual viewport)

布局视口: window.innerHeight 和 window.innerWidth 所组成的区域通常被认为是布局视口(layout viewport) window.outerHeight因为包含了浏览器header不被认为是布局视口的一部分
可视视口: 指当前浏览器中可见的部分；布局视口中超出视觉视口部分将不可见
视口：文档的 Element.clientWidth 是指一个文档使用 CSS pixels 单位表示的内部宽度, 包括其 padding (不包括 borders, margins 或垂直滚动条，如果有的话）. 这就是 viewport 的宽度。

关联：
视觉视口是屏幕的可视部分，不包括屏幕键盘，缩放外的区域。视觉视口比布局视口相同或者更小
当缩放浏览器时，布局视口改变，可视视口不变；当弹出边栏或者键盘时，布局视口不变，可视视口变小；
对于一个包含框架，objects或外部svg的页面，两者都有它们自己的window对象，只有最外层的window的视觉视口不同于布局视口。对于包含的文档，视觉视口与布局视口是相同的

在CSS中，我们也有基于视口大小的长度单位。 1vh单位是1%布局视口的高度，vw单位与此类似。

注意：当你在 CSS 中使用 vw 和 vh 设置 iframe 的样式时，1vh 表示的是 iframe 高度的1%，但 1vw 表示的则是 document 宽度的 1%。
